Ops runbook

Step-by-step admin workflow for approvals, payroll, invoices, exports, and disputes.

1) Approvals queue

  1. Open the approvals queue and set filters for the week.
  2. Search by tutor/student when needed.
  3. Select sessions and review totals.
  4. Use bulk approve or reject with a clear reason.
  5. Spot-check history for unusual changes.

2) Payroll run

  1. Open Payroll and set the week start (Monday).
  2. Generate invoices once approvals are complete.
  3. Review adjustments for the week.
  4. Lock the week after recon checks pass.

3) Invoices

  1. Confirm invoice totals and session counts.
  2. Export CSV for finance or payroll systems.
  3. Mark invoices as paid when payment is confirmed.

4) Exports

  1. Use the Audit page to export activity logs.
  2. Use Payroll CSV for week totals.
  3. Store exports with the week start date in the filename.

5) Dispute resolution

  1. Open session history and verify timestamps and notes.
  2. Check audit log for approval actions.
  3. If needed, reject the session with a clear reason.
  4. Re-run payroll if invoices were already generated.

6) Support tools

  1. Use “View as tutor (read-only)” to reproduce issues.
  2. Exit impersonation after review.
  3. Use /health and /ready for uptime checks.
